### 核心价值

**本质问题：缺乏统一的非结构化数据治理体系，导致重复建设 + 管控缺失**

| 价值点 | 解决的痛点 | 核心收益 |
| --- | --- | --- |
| 统一落点 | 各系统各自处理非结构化数据，重复造轮子 | 降低研发成本，提升效能 |
| 流向可追溯 | 数据散落各处，无法监管 | 满足审计要求，降低合规风险 |
| 统一出入口 | 多个入口，分发混乱 | 提高分发效率，统一管控 |
| 降低三方依赖 | 依赖外部平台，不可控 | 自主可控，规避业务风险 |

### 核心目标

1. **数据存储与访问**
    - 提供统一的非结构化数据上传/下载接口
    - 支持断点续传、分片上传、多存储引擎
    - 统一鉴权与权限控制
2. **生命周期治理**
    - 自动化状态流转
    - 过期文件自动回收
    - 完整的审计日志

---

### 生命周期状态机

**状态流转路径：**

```
Uploaded → Active → Expired → Recycled → Destroyed
```

| 状态 | 说明 | 触发方式 | 可执行操作 |
| --- | --- | --- | --- |
| **Uploaded** | 数据上传完成 | 用户上传 | 自动转为 Active |
| **Active** | 文件可用 | 系统自动 | 下载、删除 |
| **Expired** | 数据过期 | 定时任务扫描 | 等待回收 |
| **Recycled** | 已回收 | 系统/用户触发 | 等待物理删除 |
| **Destroyed** | 已销毁 | 定时任务 | 不可恢复 |

**关键设计：**

- **双删除来源**：区分系统自动回收（SYSTEM）和用户主动删除（USER）
- **延迟销毁**：Recycled 状态保留一定时间后才物理删除，避免误删
- **状态不可逆**：状态只能单向流转
